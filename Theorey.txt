2.prompts
In LangChain, a prompt is simply the instruction or input you give to the AI model.

It tells the AI:

what to do
how to behave
what format to answer in
Simple Example
"Explain machine learning simply"

That sentence is a prompt.

The AI reads it and generates a response.

Why Prompts Matter

The same AI can behave completely differently depending on the prompt.

Example:

Prompt 1
"Explain AI"
Prompt 2
"Explain AI to a 10-year-old using examples"

Same model.
Different output.

So prompts are basically how you “control” the AI.

Prompts in LangChain

LangChain makes prompts:

reusable
dynamic
structured

Instead of hardcoding text every time.

Basic Prompt Example
from langchain.prompts import PromptTemplate

prompt = PromptTemplate(
    input_variables=["topic"],
    template="Explain {topic} in simple words"
)

final_prompt = prompt.format(topic="Neural Networks")

print(final_prompt)

Output:

Explain Neural Networks in simple words
Why Use Prompt Templates?

Because real AI apps need dynamic prompts.

Imagine:

different users
different questions
different languages

Templates let you insert values automatically.

Suppose you build an AI tutor.

Template:

"You are a helpful teacher.
Explain {subject} to a beginner."

User enters:

LangChain creates:

"You are a helpful teacher.
Explain Python loops to a beginner."

3.Chains
In LangChain, a chain means connecting multiple AI steps together into a workflow.

Instead of doing only:

Input → AI → Output

a chain does:

Input
 ↓
Step 1
 ↓
Step 2
 ↓
Step 3
 ↓
Final Output

That’s why it’s called LangChain.

Simple Real-Life Analogy

Imagine making tea.

You don’t just:

Tea magically appears

You follow steps:

Boil water
 ↓
Add tea
 ↓
Add sugar
 ↓
Pour into cup

A chain in LangChain works similarly:
multiple connected operations.

Basic AI Example

Suppose user asks:

"Write a blog about AI"

Your chain might:

Generate title
 ↓
Generate outline
 ↓
Write article
 ↓
Summarize article

Each step feeds the next step.

Why Chains Are Useful

Real AI applications usually need:

multiple prompts
multiple models
document retrieval
formatting
reasoning steps

Chains organize all this cleanly.

Simple LangChain Example
from langchain.chains import LLMChain
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate

llm = ChatOpenAI(model="gpt-4o")

prompt = PromptTemplate(
    input_variables=["topic"],
    template="Explain {topic} simply"
)

chain = LLMChain(llm=llm, prompt=prompt)

response = chain.invoke({"topic": "Machine Learning"})

print(response)
What Happens Internally?
User Input
   ↓
Prompt Template
   ↓
LLM
   ↓
Generated Response

That whole pipeline is the chain.

More Advanced Chains

Chains can become much more powerful.

Example RAG chain:

User Question
      ↓
Convert to Embedding
      ↓
Search Vector Database
      ↓
Retrieve Relevant Chunks
      ↓
Insert into Prompt
      ↓
LLM Generates Final Answer

That is a multi-step AI workflow.

Types of Chains
1. Sequential Chains

Steps happen one after another.

Example:

Topic → Outline → Article → Summary
2. Retrieval Chains

Used in RAG systems.

They:

retrieve documents
send context to AI
generate grounded answers

Very common in:

PDF chatbots
company assistants
AI search
3. Router Chains

These decide which path to take.

Example:

If coding question → coding model
If medical question → medical chain
Chains vs Agents

This confuses many beginners.

Chains
fixed workflow
predictable steps

Example:

Step 1 → Step 2 → Step 3
Agents
dynamic decision-making
AI chooses what to do

Example:

Think
 ↓
Choose tool
 ↓
Use tool
 ↓
Analyze result

Agents are more autonomous.
